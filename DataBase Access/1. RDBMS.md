# RDBMS (Relational Database Management System)

## 📌 Definition
**RDBMS** stands for **Relational Database Management System**.  
It is a type of **DBMS** (Database Management System) that stores data in a **tabular format** (rows and columns) and maintains **relationships** between these tables.

---

## 🔑 Key Features
- **Data stored in tables**  
  Organized in rows (records) and columns (fields).
  
- **Uses SQL (Structured Query Language)**  
  For querying, updating, and managing the database.

- **Relationships between tables**  
  Established using **primary keys** and **foreign keys**.

- **Data integrity and constraints**  
  Enforces rules to keep data accurate:
  - **Primary Key** → Uniquely identifies a record.
  - **Foreign Key** → Maintains relationships between tables.
  - **Unique, Not Null, Check, Default** constraints.

- **Normalization**  
  Process of organizing data to reduce redundancy.

- **Transaction Management (ACID properties)**  
  Ensures:
  - **Atomicity**: All or nothing.
  - **Consistency**: Valid state transitions.
  - **Isolation**: Transactions don’t interfere.
  - **Durability**: Data persists after success.

---

## 🗃️ Examples of RDBMS
- Oracle Database
- MySQL
- Microsoft SQL Server
- PostgreSQL
- IBM Db2

---

## 🔍 Difference Between DBMS and RDBMS

| Feature             | DBMS                         | RDBMS                      |
|----------------------|-----------------------------|---------------------------|
| **Data Storage**     | Files                       | Tables (Relations)        |
| **Relationships**    | Not supported               | Supported (keys)          |
| **Normalization**    | Not supported               | Supported                 |
| **SQL Support**      | Limited or none             | Full SQL support          |
| **Examples**         | MS Access (Non-relational)  | MySQL, Oracle, SQL Server |

---

## 🧩 Example of RDBMS Structure

### Tables:
#### **Users**
| UserID (PK) | Name       | Email               |
|-------------|-----------|---------------------|
| 1           | Alice      | alice@email.com    |
| 2           | Bob        | bob@email.com      |

#### **Orders**
| OrderID (PK) | UserID (FK) | Product   |
|--------------|-------------|-----------|
| 101          | 1           | Laptop    |
| 102          | 2           | Phone     |

### Relationship:
- `Orders.UserID` → **Foreign Key** referencing `Users.UserID`.
