# 📘DbContext:

### ✅ Definition

`DbContext` is the **main class in Entity Framework Core (EF Core)** that acts as a bridge between your .NET application and the database.

It represents a **session with the database**, allowing you to perform operations such as:

- Querying data (Read)    
- Inserting new records (Create)    
- Updating existing records (Update)    
- Deleting records (Delete)    

---

### 🧱 Components of `DbContext`

1. **DbSet\<TEntity\>**
    Represents a table in the database.
        
```
    public DbSet<Product> Products { get; set; }
```
    
2. **Constructor**  
    Receives `DbContextOptions` that configure how the context connects to the database. This construer to receive the  configuration that put in the program.cs.

```
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }
```
    
3. **OnModelCreating Method**  
    Used to configure the model using Fluent API:
    
    - Define keys, relationships        
    - Set field lengths, types, and constraints        

```
    protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<Product>()
                .Property(p => p.Name)
                .IsRequired()
                .HasMaxLength(100);
}

```
---

### 📝 Simple Example
```
public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }

    public DbSet<Product> Products { get; set; }

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.Entity<Product>()
                    .Property(p => p.Name)
                    .IsRequired()
                    .HasMaxLength(100);
    }
}

```
### ⚙️ Registering `DbContext` in Program.cs

```
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("DefaultConnection")));

```
### 📘 Inheriting from `DbContext` in `AppDbContext`

#### ✅ What does it mean?

When you create your own context class (e.g., `AppDbContext`), you **inherit from the built-in `DbContext` class** provided by Entity Framework Core. This inheritance gives you all the core functionality needed to:

- Track entities    
- Handle database connections    
- Manage queries and updates    
- Configure models    

---

#### 🧱 Syntax Example

```
public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options)
        : base(options)
    {
    }

    public DbSet<Product> Products { get; set; }
}
```

#### 🧰 Alternative: Inheriting from a Base Context

In more complex projects, you might inherit from a **custom `BaseDbContext`** instead of directly from `DbContext`. For example:

```
public class AppDbContext : BaseDbContext
{
    public AppDbContext(DbContextOptions<BaseDbContext> options) : base(options) { }

    public DbSet<Product> Products { get; set; }
}
```

> This allows you to **reuse shared logic** like audit tracking, soft delete filters, etc., across multiple contexts.

---

#### 🚨 Important Notes

- If you're inheriting from a custom `BaseDbContext`, make sure it itself inherits from `DbContext`.
    
- When using EF tools (e.g., migrations), EF will look for a class that **inherits from `DbContext`**, either directly or indirectly.
    
- You must register your custom context in `Program.cs` using `AddDbContext<YourContext>()`.
    

---

#### 📌 Summary

> Inheriting from `DbContext` allows your custom `AppDbContext` class to interact with the database using Entity Framework Core. It's the foundation for managing database access in a clean, object-oriented way.
