#  RabbitMQ in .NET Core

## 📌 What is RabbitMQ?

RabbitMQ is a **message broker** that implements **AMQP (Advanced Message Queuing Protocol)**.  It is a messenger between applications that is responsible for communication by sending and receiving messages between microservices or components of an application.

### ✅ Why use RabbitMQ?

- **Asynchronous processing** (background tasks, email sending, payment processing)    
- **Scalability** (multiple consumers handle messages in parallel)    
- **Reliability** (messages persist until consumed)    
- **Loose coupling** between services (publish/subscribe pattern)

---

## 🔑 Key Concepts

### 1️⃣ Broker

RabbitMQ server instance where queues, exchanges, and bindings are hosted.

### 2️⃣ Queue
**A buffer that stores messages until they are consumed**. In RabbitMQ, a Queue can be referred to as temporary or buffered storage of messages, ****RabbitMQ manages a queue, where messages are stored before they are consumed by the receiver****. Queues can have the nature of durable or transient which means persisting across restarts and deleted when the server restarts respectively.

### 3️⃣ Exchange
An ****exchange**** is responsible for routing messages among queues. ****RabbitMQ provides support for various kinds of exchanges such as direct, fanout, topic, and headers, each queue with its logic.****

It supports different types of exchanges listed below

- ****Direct Exchange:**** In this exchange, messages are being routed based on routing keys. This means that when the routing key matches the binding key, the message gets delivered to its corresponding queue.
- ****Fanout Exchange:**** With this exchange, RabbitMQ does the broadcasts to all messages that are bound to it regardless of the routing key.
- ****Topic Exchange:**** With topic exchange message routing happens based on pattern matching to the routing and binding key.
- ****Headers Exchange:**** In this route messages are based on headers instead of the routing key. The headers exchange is more flexible but less efficient compared to other exchange types.

الـ **Exchange** هو المكوّن المسؤول عن **استلام الرسائل من الـ Producer**، وتحديد **كيفية توجيهها إلى الـ Queue(s)** المناسبة.

---

### 🧠 كيف يعمل؟

1. الـ **Producer** يرسل رسالة إلى **Exchange** (وليس مباشرة إلى Queue).    
2. الـ **Exchange** يستخدم **قواعد التوجيه (Routing Rules)** لتحديد أي **Queue** أو **Queues** ستستلم الرسالة.    
3. القواعد تعتمد على نوع الـ Exchange و"routing key" المرتبط بالرسالة.    

---

### 🧩 أنواع الـ Exchange في RabbitMQ:

| النوع       | الوصف                                                                      |
| ----------- | -------------------------------------------------------------------------- |
| **Direct**  | يرسل الرسائل إلى الـ Queue التي تطابق بالضبط الـ `routing key`.            |
| **Fanout**  | يرسل الرسائل **لكل** الـ Queues المربوطة به، بدون النظر للـ `routing key`. |
| **Topic**   | يسمح بتوجيه الرسائل حسب **نمط (pattern)** معين في الـ `routing key`.       |
| **Headers** | يعتمد على مطابقة **الـ headers** بدلاً من الـ `routing key`.               |

### 4️⃣ Binding & Routing Key
- In ****RabbitMQ**** a connection between queue and exchange is called ****binding****. The message route from the exchange to the queue is defined by binding based on specific criteria such as routing keys.
- The routing key plays a very important role in ****RabbitMW****, the exchange in RabbitMQ uses the routing key attribute to determine how to route the message to the appropriate queue(s). It ensures the correct message between exchanges.
![RabbitMQBinding](RabbitMQBinding.png)
### 5️⃣ Producer
A producer in RabbitMQ is an application that is responsible for sending messages to the ****RabbitMQ server****. These messages can carry various kinds of data such as [****JSON****](https://www.geeksforgeeks.org/javascript/javascript-json/), [****XML****](https://www.geeksforgeeks.org/html/xml-basics/), or simple plain text.

### 6️⃣ Consumer
The application that receives and processes messages from a queue.

### 7️⃣ Virtual Host (vhost)
Logical separation within RabbitMQ for multi-tenancy and isolation.

### 8️⃣ Connection & Channel
- **Connection**: A TCP connection to RabbitMQ.    
- **Channel**: A virtual connection over a single TCP connection (lightweight).

## 🧠How RabbitMQ Works

In the first step, the producer sends a message to RabbitMQ and the exchange receives the message. Then the exchange routes the message to one or more queues based on the routing key and exchange type. ****Once the message is in the queue, it waits for the consumer to receive and process it.**** Once the process starts, the consumer sends an acknowledgment to RabbitMQ which indicates that the message has been handled successfully. In case of no acknowledgment, RabbitMQ may resend the message to ensure the successful full process of the message.

****RabbitMQ**** provides support for multiple messaging patterns such as ****Point-to-Point****, ****Publish/Subscribe****, and ****Request/Reply.****

- ****Point-to-Point:**** In this pattern, one producer sends the message to only one queue, which will be consumed by one consumer only.
- ****Publish/Subscribe:**** In this pattern, the producer sends the message to an exchange, which routes to multiple queues. Here each queue can have one or more consumers.
- ****Request/Reply:**** Here, the consumer processes the message that it has received and sends a response back to the producer, this can be referred to as two-way communication also.

### 2️⃣ Install NuGet Packages:
> There are ways to use the rabbitMq:

#### 1️⃣ **MassTransit** (High-level abstraction)

##### 📦 **NuGet Packages to Install**

```
dotnet add package MassTransit 
dotnet add package MassTransit.RabbitMQ
```

##### ✅ **What It Does**

- Provides a **high-level abstraction** over RabbitMQ.    
- Handles:    
    - Queue/exchange creation automatically        
    - Serialization (JSON by default)        
    - Consumer registration        
    - Retries, error queues, and dead-letter queues        
    - Dependency injection with .NET        
- Supports **other transports** (Azure Service Bus, Kafka, etc.) without code changes.    

##### 💻 **Typical Usage**
```
// Register MassTransit
services.AddMassTransit(x =>
{
    x.AddConsumer<PaymentConsumer>();

    x.UsingRabbitMq((context, cfg) =>
    {
        cfg.Host("rabbitmq://localhost", h =>
        {
            h.Username("guest");
            h.Password("guest");
        });
        cfg.ReceiveEndpoint("payment_queue", e =>
        {
            e.ConfigureConsumer<PaymentConsumer>(context);
        });
    });
});

// Publisher (inside a service)
await _publishEndpoint.Publish(new OrderCreated { OrderId = 123 });
```
✅ **Best for:**
- Clean architecture    
- Built-in DI and middleware    
- Larger or scalable apps (microservices, event-driven systems)
#### 2️⃣ **RabbitMQ.Client** (Low-level client)

##### 📦 **NuGet Package to Install**

```
dotnet add package RabbitMQ.Client
```

##### ✅ **What It Does**

- Official RabbitMQ driver.    
- You manage **connections, channels, queues** manually.    
- Full control over RabbitMQ protocol and AMQP-level details.    

##### 💻 **Typical Usage**
```
var factory = new ConnectionFactory() { HostName = "localhost", UserName = "guest", Password = "guest" };
using var connection = factory.CreateConnection();
using var channel = connection.CreateModel();

channel.QueueDeclare(queue: "payment_queue", durable: false, exclusive: false, autoDelete: false);

// Publisher
var message = "Order created";
var body = Encoding.UTF8.GetBytes(message);
channel.BasicPublish(exchange: "", routingKey: "payment_queue", body: body);

// Consumer
var consumer = new EventingBasicConsumer(channel);
consumer.Received += (model, ea) =>
{
    var body = ea.Body.ToArray();
    var message = Encoding.UTF8.GetString(body);
    Console.WriteLine($"Received: {message}");
};
channel.BasicConsume(queue: "payment_queue", autoAck: true, consumer: consumer);
```
✅ **Best for:**
- Learning RabbitMQ protocol fundamentals    
- Very small/simple apps (no DI, no extra libraries)    
- Full control over AMQP without abstraction.
#### 🔥 Key Differences:

|Feature|MassTransit|RabbitMQ.Client|
|---|---|---|
|**Abstraction Level**|High-level (framework)|Low-level (driver)|
|**Setup**|Automatic (queues, exchanges)|Manual (you define all)|
|**Serialization**|Built-in JSON (configurable)|Manual (you encode/decode)|
|**DI Support**|Native with .NET DI|Manual wiring required|
|**Retries & Errors**|Built-in (error queues, retries)|Manual implementation|
|**Learning Curve**|Easier for .NET devs|Requires RabbitMQ/AMQP knowledge|

---

##### ✅ Recommendation:

- Use **MassTransit** for production apps, microservices, or if you want clean DI and automation.    
- Use **RabbitMQ.Client** if you want low-level control or are just learning AMQP basics.

## 🚀 Using RabbitMQ in .NET
there are **two approaches** to use RabbitMQ in .NET:
1. **RabbitMQ.Client** (Low-Level Manual Control)    
2. **MassTransit** (High-Level Abstraction Framework)
### 1️⃣ RabbitMQ with `RabbitMQ.Client` (Low-Level)

##### Install Package:

```
dotnet add package RabbitMQ.Client
```

---

##### 🔹 1. Basic Producer (Send to Queue)

```
using RabbitMQ.Client;
using System.Text;

var factory = new ConnectionFactory() { HostName = "localhost" };
using var connection = factory.CreateConnection();
using var channel = connection.CreateModel();

channel.QueueDeclare("demo-queue", durable: false, exclusive: false, autoDelete: false);

string message = "Hello RabbitMQ!";
var body = Encoding.UTF8.GetBytes(message);

channel.BasicPublish(exchange: "", routingKey: "demo-queue", basicProperties: null, body: body);
Console.WriteLine($"[x] Sent: {message}");
```

---

##### 🔹 2. Basic Consumer (Receive from Queue)

```
using RabbitMQ.Client;
using RabbitMQ.Client.Events;
using System.Text;

var factory = new ConnectionFactory() { HostName = "localhost" };
using var connection = factory.CreateConnection();
using var channel = connection.CreateModel();

channel.QueueDeclare("demo-queue", durable: false, exclusive: false, autoDelete: false);

var consumer = new EventingBasicConsumer(channel);
consumer.Received += (model, ea) =>
{
    var body = ea.Body.ToArray();
    var message = Encoding.UTF8.GetString(body);
    Console.WriteLine($"[x] Received: {message}");
};
channel.BasicConsume("demo-queue", autoAck: true, consumer: consumer);

Console.ReadLine();
```

---

##### 🔹 3. Exchange Types

Direct Exchange

```
channel.ExchangeDeclare("direct-exchange", ExchangeType.Direct);
channel.QueueDeclare("info-queue", false, false, false);
channel.QueueBind("info-queue", "direct-exchange", "info");

channel.BasicPublish("direct-exchange", "info", null, Encoding.UTF8.GetBytes("Info Message"));
```

Fanout Exchange

```
channel.ExchangeDeclare("logs", ExchangeType.Fanout);
channel.QueueBind("log-queue", "logs", "");
channel.BasicPublish("logs", "", null, Encoding.UTF8.GetBytes("Broadcast message"));
```

Topic Exchange

```
channel.ExchangeDeclare("topic-exchange", ExchangeType.Topic);
channel.QueueBind("payment-queue", "topic-exchange", "payment.success");
channel.BasicPublish("topic-exchange", "payment.success", null, Encoding.UTF8.GetBytes("Payment Successful"));
```

---

##### 🔹 4. Manual Acknowledgments

```
channel.BasicConsume("task-queue", autoAck: false, consumer: consumer);
consumer.Received += (model, ea) =>
{
    var msg = Encoding.UTF8.GetString(ea.Body.ToArray());
    Console.WriteLine($"Received: {msg}");
    channel.BasicAck(ea.DeliveryTag, false);
};
```

---

##### 🔹 5. Dead Letter Queues (DLX)

```
var args = new Dictionary<string, object> { { "x-dead-letter-exchange", "dlx.exchange" } };
channel.QueueDeclare("main-queue", true, false, false, args);

channel.ExchangeDeclare("dlx.exchange", ExchangeType.Fanout);
channel.QueueDeclare("dead-queue", true, false, false);
channel.QueueBind("dead-queue", "dlx.exchange", "");
```
---

##### 🔹 6. Priority Queues

```
var args = new Dictionary<string, object> { { "x-max-priority", 10 } };
channel.QueueDeclare("priority-queue", true, false, false, args);

var props = channel.CreateBasicProperties();
props.Priority = 8;
channel.BasicPublish("", "priority-queue", props, Encoding.UTF8.GetBytes("High Priority Task"));
```

### 2️⃣ RabbitMQ with **MassTransit** (High-Level)

### Install Packages:

```
dotnet add package MassTransit 
dotnet add package MassTransit.RabbitMQ
```

---

##### 🔹 1. Configure MassTransit (Startup/Program.cs)

```
builder.Services.AddMassTransit(x =>
{
    x.AddConsumer<OrderCreatedConsumer>();

    x.UsingRabbitMq((context, cfg) =>
    {
        cfg.Host("localhost", "/", h =>
        {
            h.Username("guest");
            h.Password("guest");
        });

        cfg.ReceiveEndpoint("order-created-queue", e =>
        {
            e.ConfigureConsumer<OrderCreatedConsumer>(context);
        });
    });
});
builder.Services.AddMassTransitHostedService();
```
---

##### 🔹 2. Send/Receive (Point-to-Point)

**Message Contract:**


```
public record OrderCreated(Guid OrderId, string ProductName, int Quantity);
```

**Consumer:**

```
public class OrderCreatedConsumer : IConsumer<OrderCreated>
{
    public Task Consume(ConsumeContext<OrderCreated> context)
    {
        Console.WriteLine($"Order received: {context.Message.ProductName}");
        return Task.CompletedTask;
    }
}
```

**Producer (API Controller):**

```
[HttpPost("create-order")]
public async Task<IActionResult> CreateOrder([FromServices] ISendEndpointProvider sendEndpointProvider)
{
    var endpoint = await sendEndpointProvider.GetSendEndpoint(new Uri("queue:order-created-queue"));
    await endpoint.Send(new OrderCreated(Guid.NewGuid(), "Laptop", 1));
    return Ok("Order Sent");
}
```

---

##### 🔹 3. Publish/Subscribe (Fanout)

```
await _publishEndpoint.Publish(new OrderCreated(Guid.NewGuid(), "Phone", 2));
```

✅ Any service with a consumer `IConsumer<OrderCreated>` will get the message.

---

##### 🔹 4. Topic Routing

```
cfg.ReceiveEndpoint("payment-queue", e =>
{
    e.Bind("topic-exchange", s =>
    {
        s.RoutingKey = "payment.*";
        s.ExchangeType = ExchangeType.Topic;
    });
});
```

---

##### 🔹 5. Request/Response (RPC)

```
var client = bus.CreateRequestClient<GetOrderStatus>();
var response = await client.GetResponse<OrderStatusResponse>(new GetOrderStatus(orderId));
```

---

### 🔹 6. Retry & Error Handling

```
cfg.ReceiveEndpoint("order-queue", e =>
{
    e.UseMessageRetry(r => r.Interval(3, TimeSpan.FromSeconds(5)));
    e.UseInMemoryOutbox();
});
```

---

##### 🔹 7. Delayed Scheduling

```
await _publishEndpoint.ScheduleSend(new Uri("queue:order-created-queue"),
    DateTime.UtcNow.AddMinutes(10),
    new OrderCreated(Guid.NewGuid(), "Tablet", 1));
```

---

##### 🔹 8. Dead Letter Handling

MassTransit automatically creates an `_error` queue for failed messages.

---

#### ✅ When to Use Which?

- **RabbitMQ.Client:**    
    - Fine-grained control (manual setup of exchanges/queues).        
    - Best for simple or highly customized messaging.        
- **MassTransit:**    
    - Built-in **DI, retries, error queues, scheduling, Pub/Sub**.        
    - Ideal for **enterprise/event-driven microservices**.
