> CRUD: Create Read Update Delete.

# 🔷 What is CRUD?

**CRUD** stands for the four basic operations you can perform on data in most applications:

|Operation|Meaning|HTTP Verb|Description|
|---|---|---|---|
|**C**|Create|POST|Add a new record to the database|
|**R**|Read|GET|Retrieve data (all or by ID)|
|**U**|Update|PUT/PATCH|Modify an existing record|
|**D**|Delete (or Deactivate)|DELETE|Remove or deactivate a record from the system|
# 🧭 **CRUD Flow in a .NET Project (Backend)**

## 1. **Create Operation**

- **Client** sends a POST request with data (usually JSON).    
- **Controller** receives the data (e.g., via a `ProductModel`).    
- **Service/Manager Layer** handles business logic (e.g., validation).    
- **Repository/DbContext** saves it to the database using `AddAsync()` + `SaveChangesAsync()`.    
- **Response**: Returns the created object or a success message with 201 status.

## 2. **Read Operation**

- **GET (all)**: Retrieves a list of records.    
- **GET (by ID)**: Retrieves a specific record by its ID.    
- Uses `DbContext.Products.ToListAsync()` or `FindAsync(id)`.
## 3. **Update Operation**

- **Client** sends a PUT or PATCH request with the updated data.    
- **Backend** finds the existing record using `FindAsync(id)` or LINQ.    
- Applies the updates, calls `SaveChangesAsync()`.

## 4. **Delete Operation**

- **Hard Delete**: Remove the record completely using `_context.Remove()`.    
- **Soft Delete** (preferred): Set `IsDeleted = true` and filter it from queries.
## ✅ Best Practices

- Always validate input before saving (manually or with FluentValidation).    
- Use DTOs or ViewModels to separate internal models from exposed API.    
- Implement soft deletes to preserve history.    
- Use `async/await` for performance and scalability.    
- Separate concerns into layers: Controller → Service/Manager → Repository/DbContext.
### 📦 What are **DTOs**?

**DTO** stands for **Data Transfer Object**.  
It is a **simple object** used to transfer data **between layers** of an application (especially between the API and the service or database layers).

---

### 🎯 Why use DTOs?

Here’s why we use DTOs instead of directly exposing entity classes:

|Benefit|Explanation|
|---|---|
|✅ **Security**|Prevent exposing sensitive fields (e.g., `IsDeleted`, `CreatedBy`, etc.).|
|✅ **Control**|Return only the necessary data to clients.|
|✅ **Decoupling**|Separates API models from internal database entities.|
|✅ **Validation**|Easier to validate input models without polluting entities.|
|✅ **Flexibility**|You can shape your response as needed (e.g., combine fields, rename fields).|

---

### 📥 Example: 
Input DTO (`CreateProductDto`):
```
public class CreateProductDto
{
    public string Name { get; set; }
    public decimal Price { get; set; }
    public string ProductQR { get; set; }
}
```
Output DTO (`ProductDto`):
```
public class ProductDto
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string QR { get; set; }
}
```

# Controller:

## Attributes:

| Attribute                                       | Purpose                                                                                                                              |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `[HttpGet]`                                     | Indicates the method responds to HTTP GET requests. Used to retrieve data.                                                           |
| `[HttpPost]`                                    | Indicates the method handles HTTP POST requests. Used to create new data.                                                            |
| `[HttpPut]`                                     | Handles HTTP PUT requests. Used to update an entire object.                                                                          |
| `[HttpPatch]`                                   | Handles HTTP PATCH requests. Used for partial updates.                                                                               |
| `[HttpDelete]`                                  | Handles HTTP DELETE requests. Used to delete a resource.                                                                             |
| `[Route("custom/route")]`                       | Defines a custom route for this method. Can be combined with controller-level routes.                                                |
| `[ProducesResponseType(typeof(T), StatusCode)]` | Specifies the **type** and **HTTP status code** the action returns. Helps with API documentation (e.g., Swagger).                    |
| `[FromBody]`                                    | Tells ASP.NET Core to bind this parameter from the **request body** (used for JSON inputs).                                          |
| `[FromQuery]`                                   | Binds the parameter from the **query string** (e.g., `?id=5`).                                                                       |
| `[FromRoute]`                                   | Binds the parameter from the **URL route** (e.g., `api/products/{id}`).                                                              |
| `[FromForm]`                                    | Binds from form-data (used in HTML forms or file uploads).                                                                           |
| `[FromHeader]`                                  | Binds from a request header value.                                                                                                   |
| `[Authorize]`                                   | Requires the user to be authenticated (can also specify roles or policies).                                                          |
| `[AllowAnonymous]`                              | Allows access to the action **without authentication**, even if the controller has `[Authorize]`.                                    |
| `[ValidateModel]` _(custom)_                    | Used in some projects to manually trigger model validation. ASP.NET Core does it automatically if `ApiController` attribute is used. |
## Method:
```
public async Task<ActionResult<ProductModel>> CreateProduct([FromBody] ProductModel productModel)
```

| Part                                     | Meaning                                                                                                                                                                                                                                |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `public`                                 | The method is accessible to other classes (a public API endpoint).                                                                                                                                                                     |
| `async`                                  | The method is asynchronous — it can use `await` inside, and it returns a `Task`.                                                                                                                                                       |
| `Task<...>`                              | This means the method returns a value **in the future** (asynchronously).                                                                                                                                                              |
| `ActionResult<ProductModel>`             | The HTTP **response** that will contain either:– a `ProductModel` (on success), or– an error result (e.g. 400 BadRequest, 500 Internal Server Error, etc.).                                                                            |
| `CreateProduct`                          | The name of the method (action). Follows naming convention like `CreateX`.                                                                                                                                                             |
| `([FromBody] ProductModel productModel)` | The method takes 1 parameter:– The client sends a JSON object in the **request body**.– ASP.NET deserializes it into a `ProductModel` object.– `[FromBody]` tells ASP.NET to bind this from the body (not query string, header, etc.). |
