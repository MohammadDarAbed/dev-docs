# 📦 ASP.NET Core Model Binding

## Overview

**Model Binding** is a process in ASP.NET Core that automatically maps incoming HTTP request data (such as form fields, route values, query strings, headers, and body content) to action method parameters in controllers and Razor pages.

This process simplifies working with data submitted by users and ensures your controller methods receive strongly-typed objects without requiring manual parsing.

---

## ✨ Key Features

- Automatically maps request data to .NET types.    
- Supports complex types (like classes and collections).    
- Customizable using attributes and custom binders.    
- Works seamlessly with form posts, JSON body content, route data, query strings, and more.    

---

## 🧩 How Model Binding Works

When an HTTP request is made, the model binder:

1. **Identifies the target** parameter(s) of the action method.    
2. **Searches for values** in various sources:    
    - Route values        
    - Query string        
    - Form data        
    - Headers        
    - Body (for JSON, XML, etc.)        
3. **Converts** the values to the appropriate .NET type.    
4. **Assigns** them to the action parameters.    

---

## 🗂 Supported Sources

|Source|Description|
|---|---|
|Route|Values embedded in the URL pattern|
|Query String|Data passed in the URL after `?`|
|Form Data|Data posted from HTML forms (application/x-www-form-urlencoded or multipart/form-data)|
|Body|JSON/XML data sent with application/json|
|Header|Data in the HTTP headers|

---

## 🧪 Example: Simple Binding

```
[HttpGet("product/{id}")]
public IActionResult GetProduct(int id, string name)
{
    // id comes from route, name from query string
    return Ok($"Product ID: {id}, Name: {name}");
}
```
- URL: `/product/5?name=chair`    
- Output: `"Product ID: 5, Name: chair"    

---

## 🧱 Binding to Complex Types

```
public class ProductDto
{
    public int Id { get; set; }
    public string Name { get; set; }
}

[HttpPost]
public IActionResult CreateProduct(ProductDto product)
{
    // ASP.NET Core binds form or JSON data to the ProductDto object
    return Ok(product);
}
```

> Data can come from JSON body (with `application/json`) or form fields.

---

## 📌 Binding Attributes

Use attributes to control where the data comes from:

|Attribute|Source|
|---|---|
|`[FromQuery]`|Query string|
|`[FromRoute]`|Route values|
|`[FromForm]`|Form data|
|`[FromBody]`|Request body (e.g. JSON)|
|`[FromHeader]`|HTTP headers|
|`[FromServices]`|Dependency injection|

```
public IActionResult UpdateProduct([FromBody] ProductDto product)
```

---

## ⚠️ Binding Limitations

- `[FromBody]` can only be used once per action method.    
- Model binding does not run on static classes or parameters without setters.    
- If binding fails, the parameter is null or has a default value.

## ✅ Example Controller with All Model Binding Attributes

```
[ApiController]
[Route("api/[controller]")]
public class ExampleController : ControllerBase
{
    // [FromQuery]: gets data from the URL query string
    [HttpGet("fromquery")]
    public IActionResult GetFromQuery([FromQuery] string name)
    {
        return Ok($"FromQuery: {name}");
    }

    // [FromRoute]: gets data from the route pattern
    [HttpGet("fromroute/{id}")]
    public IActionResult GetFromRoute([FromRoute] int id)
    {
        return Ok($"FromRoute: {id}");
    }

    // [FromForm]: gets data from an HTML form (POST request)
    [HttpPost("fromform")]
    public IActionResult PostFromForm([FromForm] string email)
    {
        return Ok($"FromForm: {email}");
    }

    // [FromBody]: gets data from the body (e.g. JSON)
    public class PersonDto
    {
        public string FirstName { get; set; }
        public int Age { get; set; }
    }

    [HttpPost("frombody")]
    public IActionResult PostFromBody([FromBody] PersonDto person)
    {
        return Ok($"FromBody: {person.FirstName}, Age: {person.Age}");
    }

    // [FromHeader]: gets data from the HTTP headers
    [HttpGet("fromheader")]
    public IActionResult GetFromHeader([FromHeader(Name = "X-Custom-Header")] string headerValue)
    {
        return Ok($"FromHeader: {headerValue}");
    }

    // [FromServices]: gets a service from the DI container
    [HttpGet("fromservices")]
    public IActionResult GetFromServices([FromServices] ITimeService timeService)
    {
        return Ok($"FromServices: {timeService.GetCurrentTime()}");
    }
}
```
## 🧪 Example Requests

| Attribute        | Request Example                                                                                                                                 |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `[FromQuery]`    | `GET /api/example/fromquery?name=Ali` → returns `"FromQuery: Ali"`                                                                              |
| `[FromRoute]`    | `GET /api/example/fromroute/123` → returns `"FromRoute: 123"`                                                                                   |
| `[FromForm]`     | `POST /api/example/fromform` with `Content-Type: application/x-www-form-urlencoded` and body: `email=test@example.com` → `"FromForm: test@..."` |
| `[FromBody]`     | `POST /api/example/frombody` with `Content-Type: application/json` and body: `{"firstName": "Sara", "age": 25}` → `"FromBody: Sara, Age: 25"`   |
| `[FromHeader]`   | `GET /api/example/fromheader` with header `X-Custom-Header: 999` → returns `"FromHeader: 999"`                                                  |
| `[FromServices]` | Automatically injects the service (see below) and returns result like `"FromServices: 2025-06-30 12:00:00"`                                     |
