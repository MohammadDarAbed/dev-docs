## 📌 APIs in .NET

### 🔹 What is an API?

- **API (Application Programming Interface)**: A contract that allows two applications to communicate with each other.    
- In .NET, APIs are commonly **HTTP-based (RESTful APIs)** built using **ASP.NET Core**.    

---

### 🏗 Types of APIs in .NET

1. **Controller-based API (Traditional)**    
    - Uses **Controllers** that inherit from `ControllerBase`.        
    - Organizes endpoints in separate controller classes.        
    - Example:

```
[ApiController]
[Route("api/[controller]")]
public class ProductsController : ControllerBase
{
    [HttpGet]
    public IActionResult GetProducts() => Ok(new[] { "Product1", "Product2" });
}
```

2. **Minimal API (New in .NET 6+)**    
    - Lightweight way of creating APIs without controllers.        
    - Define endpoints directly in `Program.cs`.        
    - Ideal for **microservices** or **small APIs**.        

---

### 🔑 REST API Principles

- **Stateless**: Each request is independent.    
- **Resource-based**: Exposes resources (e.g., `/products`).    
- **CRUD Operations**:    
    - `GET`: Retrieve data.        
    - `POST`: Create new resource.        
    - `PUT/PATCH`: Update existing resource.        
    - `DELETE`: Remove resource.        

---

## ⚡ Minimal APIs

### 🔹 What is a Minimal API?

- Introduced in **.NET 6**.    
- **Definition**: A simplified API programming model for small, fast HTTP APIs with minimal boilerplate code.    
- **Goal**: Reduce ceremony (no controllers, no attributes, fewer files).    

---

### ✨ Features of Minimal APIs

✅ No Controllers → Endpoints are defined inline.  
✅ Lightweight & Fast → Lower overhead than MVC.  
✅ Perfect for **microservices, serverless APIs, or prototyping**.  
✅ Supports **dependency injection**, **filters**, **middleware**, and **OpenAPI (Swagger)**.

---

### 🛠 Minimal API Example

```
var builder = WebApplication.CreateBuilder(args);
var app = builder.Build();

// GET endpoint
app.MapGet("/hello", () => "Hello, World!");

// POST endpoint with input
app.MapPost("/greet", (User user) => $"Hello {user.Name}");

app.Run();

record User(string Name);
```

---

### 🔗 Binding Data in Minimal APIs

- Automatically binds from:    
    - **Query strings**        
    - **Route parameters**        
    - **Request body (JSON)**        
    - **Form data**        

Example:

```
app.MapGet("/products/{id}", (int id) => $"Product ID: {id}");
app.MapPost("/products", (Product p) => $"Added: {p.Name}");
```

---

### 🧩 Dependency Injection in Minimal APIs

```
builder.Services.AddScoped<IProductService, ProductService>();

app.MapGet("/products", (IProductService service) =>
{
    return service.GetAllProducts();
});
```

---

### 🛡 Authentication & Authorization in Minimal APIs


```
app.MapGet("/secure", [Authorize] () => "Secure Data");
```

---

### 📦 Adding Swagger for Minimal APIs

```
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

app.UseSwagger();
app.UseSwaggerUI();
```

---

### 🆚 Minimal API vs Controller API

|Feature|Minimal API|Controller API|
|---|---|---|
|Boilerplate|Very low|Higher (Controllers, Attributes)|
|Complexity|Simple projects|Large/Complex projects|
|Flexibility|Less (inline)|More (filters, MVC features)|
|Performance|Faster (less overhead)|Slightly slower|

---

### 🔥 When to Use Minimal APIs?

✅ Small microservices  
✅ Prototypes or MVPs  
✅ Serverless functions  
✅ Simple CRUD APIs

❌ For complex apps → Prefer **Controller-based APIs**.

---

## 🚀 Best Practices for APIs (Both Controller & Minimal)

- ✅ Use **DTOs (Data Transfer Objects)** to avoid exposing entities.
    
- ✅ Validate inputs with `FluentValidation` or `DataAnnotations`.
    
- ✅ Enable **Swagger/OpenAPI** for documentation.
    
- ✅ Handle **global error handling** with middleware.
    
- ✅ Use proper **HTTP status codes** (`200 OK`, `400 BadRequest`, `404 NotFound`).
