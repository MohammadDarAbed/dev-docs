## 🧩 DRY (Don't Repeat Yourself) Principle

### 📖 Definition

> **Every piece of knowledge must have a single, unambiguous, authoritative representation within a system.**

In other words:

- Avoid **code duplication**.    
- Extract repeating logic into **reusable functions, classes, or modules**.    

---

### 🚩 **Why DRY?**

- Reduces maintenance cost: **Fix bugs in one place, not many.**    
- Makes code cleaner and more readable.    
- Prevents inconsistencies caused by duplicated logic.    

---

### 🛑 **Violation of DRY**

A common example: **calculating discounts in multiple places**.
```
public class OrderService
{
    public decimal CalculateFinalPrice(decimal price)
    {
        decimal discount = 0;

        // Discount logic duplicated here
        if (price > 100) discount = price * 0.10m;

        return price - discount;
    }
}

public class InvoiceService
{
    public decimal GenerateInvoice(decimal price)
    {
        decimal discount = 0;

        // Same discount logic repeated again!
        if (price > 100) discount = price * 0.10m;

        return price - discount;
    }
}
```
**Problems:**

- Logic is duplicated across multiple classes.    
- If the discount rule changes (e.g., 15%), you must update it in every place.    

---

### ✅ **Applying DRY**

Extract the shared logic into a **single reusable method/class**.
```
public class DiscountService
{
    public decimal ApplyDiscount(decimal price)
    {
        if (price > 100) return price * 0.90m; // 10% discount
        return price;
    }
}
```
Now reuse it in other classes:
```
public class OrderService
{
    private readonly DiscountService _discountService = new DiscountService();

    public decimal CalculateFinalPrice(decimal price)
    {
        return _discountService.ApplyDiscount(price);
    }
}

public class InvoiceService
{
    private readonly DiscountService _discountService = new DiscountService();

    public decimal GenerateInvoice(decimal price)
    {
        return _discountService.ApplyDiscount(price);
    }
}
```
✅ **Single source of truth:**  
If the discount changes to 15%, update **only `DiscountService`**.

---

### 🔑 Real-World Examples

- **Validation logic:** Centralize validation rules instead of repeating them across forms/controllers.  
- **Date formatting:** Use a shared utility method instead of rewriting `ToString("yyyy-MM-dd")` everywhere.    
- **API calls:** Centralize HTTP request code in a service instead of duplicating fetch logic.    

---

### 💡 Key Takeaways

- Identify **duplicate logic** and extract it.    
- Use **utility classes, shared services, or extension methods** for reusability.    
- DRY improves **maintainability, readability, and consistency**.
