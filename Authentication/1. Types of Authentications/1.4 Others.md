# 🔐 Common Authentication Types

|Authentication Type|Description|Use Cases|Security Level|
|---|---|---|---|
|**Basic Authentication**|Sends Base64 encoded username & password in HTTP header|Legacy APIs, simple internal apps|Low (must use HTTPS)|
|**Bearer Authentication**|Sends access token (often JWT) in Authorization header|Modern APIs, OAuth2-based systems|Medium to High|
|**Digest Authentication**|Challenge-response mechanism hashing credentials|Some older web apps|Medium (better than Basic)|
|**OAuth 2.0**|Delegated authorization framework issuing tokens|Third-party API access, social login|High|
|**OpenID Connect (OIDC)**|Authentication layer on top of OAuth 2.0|Single Sign-On (SSO)|High|
|**API Key**|Static secret key sent with requests|Simple service-to-service auth|Low to Medium|
|**SAML (Security Assertion Markup Language)**|XML-based protocol for SSO and identity federation|Enterprise SSO|High|
|**Certificate-Based Authentication**|Uses client SSL/TLS certificates for authentication|High-security environments|Very High|
|**Windows Authentication (NTLM/Kerberos)**|Uses Windows domain credentials|Internal corporate networks|High|
|**JWT (JSON Web Token)**|Token-based authentication (used with Bearer)|Stateless authentication|Medium to High|
## Quick Descriptions

### Basic Authentication
- Simple but insecure without HTTPS.    

### Bearer Authentication
- Uses tokens (JWT or OAuth tokens).    
- Secure when combined with HTTPS and token expiration.    

### Digest Authentication
- Sends hashed credentials, better than Basic but less used today.    

### OAuth 2.0
- Industry-standard for delegated authorization.    
- Users authorize apps to access their data without sharing passwords.    

### OpenID Connect (OIDC)
- Adds authentication layer to OAuth 2.0.    
- Enables Single Sign-On (SSO) and identity info.    

### API Key
- Simple secret key often in headers or URL.    
- Easy but less secure, lacks user identity context.    

### SAML
- XML-based standard for exchanging authentication and authorization data.    
- Common in enterprises for SSO.    

### Certificate-Based Authentication
- Client proves identity using SSL/TLS certificate.    
- Very secure but complex to manage.    

### Windows Authentication
- Uses Windows domain accounts.    
- Common in intranet applications.    

---

## Summary

- **Choose authentication based on your app needs:**    
    - Simple & internal → Basic or API key.        
    - Public APIs → OAuth2 / Bearer tokens.        
    - Enterprise → SAML or OIDC.        
    - High-security → Certificates or Kerberos.
